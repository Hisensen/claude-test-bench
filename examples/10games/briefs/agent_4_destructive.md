# 测试简报 — Agent 4：破坏性测试员

## 你的角色
你是一名**专门寻找边界案例和程序崩溃点**的测试专家。你的目标不是正常使用游戏，而是通过非预期操作找到 bug、崩溃或行为异常。你会使用所有可能让程序出错的手段。

## 项目概述
被测项目是一个包含 10 款小游戏的网页（地址见运行时信息）。

## 测试方法
读取 index.html 源码，对每个游戏的 JavaScript 逻辑进行**代码级静态审查**，寻找：
- 边界值处理缺失
- 竞态条件（race condition）
- 状态重置不完整
- 全局变量污染
- 事件监听器泄漏
- 定时器未清理

也可以用 Playwright 模拟破坏性操作。

## 破坏性测试清单

### 全局结构
- [ ] 游戏之间是否存在全局变量冲突（window.xxx 是否在关闭游戏时被正确删除）？
- [ ] 快速连续打开/关闭不同游戏，是否会出现定时器叠加（两个游戏同时运行）？
- [ ] 点击覆层外部关闭游戏，再重新打开，状态是否正确重置？

### 猜数字
- [ ] 输入负数、0、101、999，是否有校验？
- [ ] 输入小数（如 3.5）会怎样？
- [ ] 成功后强行继续输入，是否有异常？

### 井字棋
- [ ] 游戏结束后再点格子，是否会出错？
- [ ] AI 落子期间（300ms 延迟）玩家快速点击，是否会出现双 X？
- [ ] 平局判断：棋盘满格时是否总能正确识别？

### 记忆翻牌
- [ ] 在翻牌动画期间（850ms）快速点第三张牌，会怎样？
- [ ] 快速点击同一张牌两次，是否被正确处理？
- [ ] 所有配对完成后，`matched.size` 是否恰好等于 16？

### 打地鼠
- [ ] 游戏结束后地鼠洞是否残留地鼠（textContent 未清空）？
- [ ] 快速连击同一个洞，得分是否会超加？
- [ ] 多次点击"开始"，是否会叠加多个 setInterval？

### 反应速度
- [ ] 测试完 5 轮后，再次点击是否正常重新开始？
- [ ] "太早了"后立即再次点击，逻辑是否正确？

### 数学闪题
- [ ] 输入非数字字符时是否有防护？
- [ ] 题目切换瞬间快速提交，是否会判断上一题的答案？

### 颜色记忆
- [ ] 序列播放过程中点击按钮，是否会中断播放？
- [ ] 同时快速点击多个按钮，`usr` 数组是否会错乱？

### 贪吃蛇
- [ ] 快速连续按反向键（右→左），是否会让蛇立即反向穿越自身？
- [ ] 多次点击"开始/重置"，是否会叠加多个 setInterval？
- [ ] 关闭游戏时，键盘监听器是否被正确移除？

### 弹球打砖
- [ ] 多次点击"开始/重置"是否叠加定时器？
- [ ] 鼠标快速移到 canvas 边缘，挡板是否会超出边界？
- [ ] 关闭游戏后鼠标移动，mousemove 监听器是否已被移除？

### 扫雷
- [ ] 右键插旗后再左键点击，是否正确拦截？
- [ ] 游戏结束后继续点击，是否有异常？
- [ ] 边角格子的 flood-fill 递归，是否有栈溢出风险（9×9 应该没问题，但需确认）？
